Agentic System Architecture

Roblox Runtime

visual + input
Roblox Clients

Players interact with the cell world through UI, movement, inspection, and tools.

RemoteEvents

Clients talk only to the Roblox server, never directly to the backend.

Roblox Server Bridge

Samples object attributes and sends batched observations over HTTPS.

Visual Delta Applier

Receives approved deltas: color, glow, animation, label keys, and cues.

>

Backend Authority

source of truth
Session Coordinator

One active simulation context per Roblox server or game room.

Object Registry

Tracks object ids, archetypes, parameter schemas, ownership, and visibility.

Agent Runtime

Maintains backend agent records: role, status, memory, task, cooldowns.

Deterministic Simulation

Runs biology and game rules at the fast backend tick.

Validator + Reducer

Accepts, clamps, or rejects proposed actions before state changes.

Delta Builder

Queues compact updates for Roblox polling or long-polling.

>

LLM Game Director

advisory
Compressed World Summary

Receives only compact state, not the full Roblox Workspace.

Object Groups

Reasons over groups like low-oxygen mitochondria, not every part.

Structured Directives

Returns schema-constrained intent, not raw Lua code.

Teaching Moments

Chooses scenario priorities, explanations, and learning prompts.

Safety Boundary

Every directive goes back through backend validation before use.

Roblox renders and collects input Backend owns canonical state LLM directs, but does not write state

System And Data Flow

1Sample

Roblox samples object state and player actions.

2Observe

Server bridge sends a batched observation packet.

3Authenticate

Backend checks token, session, schema, and source.

4Update State

Canonical state and event log are updated.

5Simulate

Deterministic rules advance the world state.

6Direct

LLM gets a compact summary only when needed.

7Validate

Directives are accepted, clamped, or rejected.

8Queue

Backend queues approved visual deltas.

9Apply

Roblox polls and applies approved deltas.

Final Architecture Decision

Build now: FastAPI Backend Authority

Use batched HTTPS observations from Roblox, deterministic simulation in FastAPI, backend agent records per object, optional LLM game director, strict validation, and queued visual deltas back to Roblox.

Do not build yet

No direct Roblox-to-LLM calls, no one-LLM-agent-per-object architecture, no WebSocket requirement for Roblox gameplay, and no Cloudflare Durable Object layer until the MVP proves traffic and gameplay needs.
